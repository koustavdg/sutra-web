Changelog

Every Sutra release, one thread at a time — what's new, what's improved, and what's fixed.

1.3
Shipped

Reading features and server compatibility — text highlights with iCloud sync, instant word definitions, series-based organization, plus Calibre-Web-Automated support and improved compatibility for Tailscale, Grimmory, and other OPDS servers.

New
  • Text Highlights: Select text while reading (EPUB or PDF), choose a color, and the passage stays highlighted in the book and automatically saved to Settings → Text Highlights. Tap a highlight to change its color or delete it. Highlights sync via iCloud across your devices, just like reading progress, and can be browsed from the reader's Bookmarks sheet.
  • Dictionary lookup: Double-tap a word while reading (EPUB or PDF) to see its definition immediately, with one-tap copy — powered by the free Dictionary API (dictionaryapi.dev). Every word you look up is saved to Settings → Dictionary for later reference.
  • Collections by series: Books now organize by series in the Collections Shelf, based on metadata from your library server — brings series browsing to Calibre, Calibre-Web, and self-hosted OPDS servers.
  • Support for Calibre-Web-Automated (CWA), a community fork with automatic ebook ingestion — connect it the same way as Calibre-Web (default port 8084).
Improved
  • Server detection now rejects non-OPDS responses (e.g., HTML login portals from reverse proxies or single-sign-on) instead of silently accepting an empty library.
  • Grimmory shelves (and Calibre-Web collections) now appear as read-only collections in the Shelf tab under "Server Shelves" — these update automatically when you refresh your library.
  • Tailscale MagicDNS names (e.g., myserver.tailnet-name.ts.net) now connect over plain HTTP like any other self-hosted server.
  • Empty libraries (books all in unsupported formats like MOBI, AZW3, CBZ) now show a clear message with conversion tips instead of appearing as a silent blank shelf.
  • Newly added servers now auto-retry if they report "unavailable" immediately after granting the Local Network permission, instead of requiring manual pull-to-refresh.
  • Cover-image cache increased to 1500 images / 200 MB (up from 200 / 64 MB), with prefetching for Continue Reading.
Fixed
  • Online search now works against self-hosted OPDS servers that Sutra doesn't specifically recognize (Komga, Grimmory, Booklore, BookOrbit, OPDShelf) when the server advertises search capability.
  • Series information from self-hosted servers tagged like Calibre now reads more reliably instead of occasionally showing a genre as the series name.
  • Library screen no longer shows nothing for several seconds on launch — in-progress books you've downloaded now appear in Continue Reading instantly.
1.2
Shipped

A big reliability and polish update — better cross-device sync, a real page-curl animation, dark mode, and a wide pass of fixes across search, downloads, and the reader.

New
  • Authentic page-curl page turns — tap or drag to peel a page like a real book (turn it off for the classic instant/slide turn in Settings → Accessibility)
  • Appearance: Day, Night, or System — a real dark theme for the whole app; book pages keep their own reading theme
  • Left-Handed Page Turns — tap the left edge to go to the next page, right edge to go back
  • Series info and sorting from Kavita libraries — powers the "More in this series" carousel and a proper Series sort
Improved
  • Standard Ebooks now connects with your Patrons Circle email, verified against the live feed before the library is added
  • Self-hosted servers on your local network with no TLS now connect automatically
  • Paged vs. continuous scrolling is remembered separately for EPUB and PDF
  • "Recently Read" and Continue Reading now rank by when you actually read, not raw progress
Fixed
  • Large self-hosted libraries (Kavita, Komga) could silently lose books partway through loading, so those books never turned up in search — big libraries now load completely
  • Kavita reading position now syncs correctly in both directions for EPUBs
  • Cross-device iCloud progress sync now works reliably for every connected library
  • Books no longer get marked "Finished" from a glance at the last page
  • Fixed a case where a server's feed could confuse two different books with each other
  • The library no longer goes blank after a brief server hiccup
  • Hardened how credentials and cover images are handled
  • A range of smaller fixes across search, downloads, and the readers
